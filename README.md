# 视频管理系统

一个面向移动端优化的本地视频管理系统，基于 Flask + 原生前端实现。当前版本重点解决了旧版首屏过重、手机发热、缩略图比例混乱、信息冗余和文档失效的问题。

## 当前版本特点

- 移动端优先：首屏分页加载，滚动增量渲染，避免一次性渲染全部媒体
- 轻量播放：列表页只展示缩略图，播放时使用单实例播放器
- 统一卡片结构：缩略图、文件名、标签、大小、重命名、删除
- 缩略图优化：统一裁剪容器，服务端按需生成压缩缩略图缓存
- 分类浏览：支持按标签过滤，适合手机快速滑看
- 文件操作安全：路径做安全拼接，删除为软删除，重命名限制在原目录内
- 便于维护：模板、静态资源、依赖和 README 已按当前版本重构

## 运行数据目录

默认情况下，系统使用下面这些目录：

- 视频目录：`D:\static`
- 删除回收目录：`D:\delete`
- 缩略图缓存目录：`D:\templates\cache\thumbs`

也可以通过环境变量自定义：

- `VIDEO_LIBRARY_DIR`
- `VIDEO_DELETE_DIR`
- `VIDEO_THUMB_CACHE_DIR`

## 媒体组织规则

系统会递归扫描视频目录下的所有子目录，当前支持：

- 视频：`.mp4`
- 缩略图：`.jpg` `.jpeg` `.png` `.webp`

推荐命名：

```text
<标签> 文件名.mp4
<标签> 文件名.jpg
```

示例：

```text
D:\static\电影\动作 极速车王.mp4
D:\static\电影\动作 极速车王.jpg
```

标签提取规则：

- 优先取文件名第一个空格前的内容
- 会自动移除 `#`
- 如果文件名前缀不可用，则回退到顶层文件夹名

## 安装与启动

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动服务

```bash
python app.py
```

默认监听：

```text
http://127.0.0.1:5000
```

同一局域网内的手机可访问：

```text
http://你的电脑IP:5000
```

## 前端交互说明

- 卡片只展示必要信息：标签、文件大小
- 文件名单行截断，可直接点击重命名
- 缩略图统一为固定比例容器，避免横竖图导致布局抖动
- 支持无限滚动和手动“加载更多”
- 删除为软删除，文件会移动到删除目录

## API 概览

### `GET /`

返回管理页面。

### `GET /api/videos`

分页获取视频列表。

查询参数：

- `page`
- `page_size`
- `category`

返回内容包括：

- `items`
- `total`
- `total_pages`
- `has_more`
- `stats`
- `filters`

### `GET /api/categories`

返回分类列表及每个分类下的视频数量。

### `POST /api/rename`

重命名视频及同名缩略图。

请求体：

```json
{
  "oldName": "目录/旧文件名.mp4",
  "newName": "目录/新文件名.mp4"
}
```

### `POST /api/delete`

软删除视频及同名缩略图。

请求体：

```json
{
  "name": "目录/文件名.mp4"
}
```

### `GET /media/<path>`

返回原始视频流，支持浏览器范围请求。

### `GET /thumbnail/<path>?w=480`

返回压缩缩略图，带缓存。

## 性能策略

当前版本为了解决旧版手机发热问题，采用了以下策略：

- 服务端分页，避免一次返回全部视频
- 前端增量渲染，避免超大 DOM
- 图片懒加载，只在接近视口时加载
- 列表页不嵌入多实例视频播放器
- 服务端生成压缩缩略图并缓存，避免手机解码原图

## 已知注意事项

- 当前视频格式仅支持 `.mp4`
- 如果未安装 `Pillow`，缩略图接口会回退到原图输出，移动端性能会变差

## 建议的后续增强

- 接入 `ffprobe` 或媒体元数据索引，服务端直接提供时长
- 增加批量操作能力
- 增加封面缺失时的自动首帧截图
- 增加登录保护或局域网访问限制
